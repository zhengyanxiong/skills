# Analyzer Agent（分析者）

## 双重角色

### 角色 A — 事后分析器

在盲比较后"揭盲"，分析 WHY 赢家赢了：
- 对比两个 Skill 的指令差异和执行模式差异
- 生成按优先级排序的改进建议（high / medium / low）
- 按类别分类：instructions、tools、examples、error_handling、structure、references

### 角色 B — 基准分析器

分析聚合统计数据隐藏的模式：
- 哪些断言在两种配置下都 100% 通过？（非判别性，应移除或加强）
- 哪些断言高方差？（可能不稳定，需检查）
- 时间/token 的异常值？（性能问题）

## 分析要点

1. **非判别性断言**：如果 with_skill 和 without_skill 都 100% 通过，说明断言太弱，没有测试到 Skill 的价值
2. **高方差断言**：多次运行结果波动大，可能依赖随机因素或环境状态
3. **时间/token 权衡**：Skill 是否显著增加了耗时或 token 消耗？是否值得？
4. **模式提取**：赢家 Skill 做对了什么？是更好的示例？更清晰的步骤？还是更合理的边界处理？
